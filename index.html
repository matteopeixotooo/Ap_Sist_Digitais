O Problema Técnico e Vetor de Teste

Slide 2 / 18

🎯 Escopo do Desafio

Desenvolver um software em Assembly e adaptar uma arquitetura de hardware monociclo para ler, analisar e extrair extremos numéricos contidos em um array na RAM.

📥 Estado Inicial da Memória RAM

A memória foi populada consecutivamente com 8 elementos inteiros de 32 bits a partir do endereço base zero:

[15, 42, 4, 89, 23, 7, 55, 12]

📤 Metas Logísticas de Execução

  • Varrer as posições da RAM sem deixar janelas ociosas.
  • Isolar dinamicamente o maior valor do vetor (89).
  • Isolar dinamicamente o menor valor do vetor (4).
  • Garantir a persistência salvando o 89 na posição 32 e o 4 na posição 36 da RAM ao fim da varredura.

Programa Assembly MIPS Desenvolvido

Slide 3 / 18
-- Rotina de Inicialização
addi $t0, $zero, 0   -- Ponteiro de endereço da RAM = 0
addi $t1, $zero, 32  -- Limite de parada (8 elementos * 4 bytes)
lw   $t2, 0($t0)     -- MAX provisório inicial = Vetor[0] (15)
lw   $t3, 0($t0)     -- MIN provisório inicial = Vetor[0] (15)

-- Laço de Repetição Principal (Loop)
addi $t0, $t0, 4     -- Avança o ponteiro para o próximo endereço
beq  $t0, $t1, FIM   -- Se ponteiro == 32, desvia para o encerramento
lw   $t5, 0($t0)     -- Carrega o elemento atual da RAM para $t5
max  $t2, $t2, $t5   -- Nova Instrução: Atualiza o MAX se $t5 > $t2
min  $t3, $t3, $t5   -- Nova Instrução: Atualiza o MIN se $t5 < $t3
j    16              -- Salto incondicional para o início do laço

-- Rotina de Salvamento e Encerramento
FIM:
addi $s0, $zero, 32  -- Registrador base de escrita fixado em 32
sw   $t2, 0($s0)     -- Escreve o valor do MAX (89) na RAM[32]
sw   $t3, 4($s0)     -- Escreve o valor do MIN (4) na RAM[36]
j    52              -- Loop infinito de segurança (Trava o PC)

💡 Análise do Código de Software

O programa divide-se de forma clara em três etapas fundamentais:


  • Preparação: Configura os registradores base de controle e espelha o primeiro elemento nos dois acumuladores.
  • Varredura: Atualiza as variáveis usando as instruções de hardware customizadas em apenas um ciclo de clock.
  • Gravação: Salva os extremos calculados de volta na memória e congela o Program Counter.

Arquitetura de Hardware: Módulo Top-Level

Slide 4 / 18

🏢 Entidade maxmin_final

Funciona como a infraestrutura central do processador. O seu papel é instanciar e interconectar os barramentos dos componentes principais:


  • Memória de Instruções (ROM): Barramento combinacional estável de leitura do programa.
  • Banco de Registradores: Armazenamento temporário de dados e registradores de debug.
  • ALU Estendida: Unidade matemática com suporte aos novos operadores lógicos.
entity maxmin_final is
  Port (
    clk            : in  STD_LOGIC;
    reset          : in  STD_LOGIC;
    out_pc         : out STD_LOGIC_VECTOR(31 downto 0);
    out_alu_res    : out STD_LOGIC_VECTOR(31 downto 0);
    out_opcode     : out STD_LOGIC_VECTOR(5 downto 0);
    out_ALUControl : out STD_LOGIC_VECTOR(2 downto 0);
    out_WD         : out STD_LOGIC_VECTOR(31 downto 0);
    out_MemWrite   : out STD_LOGIC
  );
end entity;

Sincronismo do Caminho de Dados

⚠️ Elementos Síncronos (Dependem do Clock)

Apenas os blocos que alteram o estado físico do hardware reagem de forma síncrona na borda de subida do clock:


  • O registrador do Program Counter (PC).
  • A gravação de dados dentro das células do Banco de Registradores.
  • A persistência e gravação final de variáveis na Memória RAM.

⚡ Elementos Combinacionais (Imediatos)

Fluem de maneira contínua através dos fios lógicos, atualizando as saídas instantaneamente ao mudar as entradas:


  • A busca das instruções na memória ROM.
  • A decodificação de comandos na Unidade de Controle.
  • Os cálculos e testes de comparação internos da ALU.
  • A leitura assíncrona de dados armazenados na RAM.

Extensão da Unidade de Controle

Slide 6 / 18
case opcode is
  when "000000" => -- Tipo-R Geral
    RegWrite <= '1'; RegDst <= '1';
    case funct is
      when "100000" => ALUControl <= "010"; -- add
      when "000101" => ALUControl <= "011"; -- max
      when "000110" => ALUControl <= "100"; -- min
      when others   => ALUControl <= "000";
    end case;
end case;

🧠 Decodificação por Nível Aninhado

Mantivemos as novas instruções sob a raiz do formato Tipo-R para preservar a compatibilidade nativa da arquitetura:


  • A unidade lê o Opcode geral de zeros "000000".
  • Ativa as flags de permissão de escrita RegWrite e seletor de destino RegDst.
  • Um segundo switch interno de controle analisa o campo Funct para direcionar o código operacional correto para a ALU.

Modificação Interna da ALU em VHDL

Slide 8 / 18

🛠️ Lógica de Comparação com Sinal

Injetamos dois novos blocos operacionais baseados em comparadores condicionais com sinal (signed) para processar os barramentos A e B:


  • Se ALUControl for "011", a ALU ativa a diretiva de máximo e direciona o maior valor para a saída.
  • Se ALUControl for "100", a ALU ativa a diretiva de mínimo e direciona o menor valor para a saída.
case ALUControl is
  when "011" => -- Operação MAX
    if signed(A) > signed(B) then 
      resultado <= A;
    else 
      resultado <= B; 
    end if;
  when "100" => -- Operação MIN
    if signed(A) < signed(B) then 
      resultado <= A;
    else 
      resultado <= B; 
    end if;
end case;

Cálculo do Salto Relativo (beq): O campo de imediato armazena o valor inteiro 4. Em hardware, a CPU calcula o endereço alvo aplicando a fórmula: Endereço Target = PC_plus4 + (imediato * 4) = 24 + (4 * 4) = 40 (0x28). Isso faz o fluxo pular o laço perfeitamente quando o vetor termina.

Mapeamento do Salto Absoluto Final (j 13): Para travar a CPU no endereço físico 52 (0x34), o campo target armazena o valor absoluto dividido por 4 (52 / 4 = 13, ou seja, 0x0D em bits). O hardware desloca 2 bits à esquerda e reconstrói o alvo fixo, impedindo falhas de execução.

Alinhamento Little-Endian na Memória ROM

Slide 11 / 18

🔄 Inversão Física de Bytes

A entidade memoria_instrucoes foi modelada fisicamente como uma matriz linear de bytes isolados de 8 bits. Para respeitar o padrão Little-Endian, o byte menos significativo (LSB) deve ocupar o menor endereço de índice:


Por exemplo, a instrução 0x21080004 (PC = 16) é mapeada nos slots contíguos de forma invertida: slot 16 recebe X"04", slot 17 recebe X"00", slot 18 recebe X"08" e o slot 19 recebe X"21".

architecture rtl of memoria_instrucoes is
begin
  idx <= to_integer(unsigned(A(6 downto 0)));

  -- Reconstrução Combinacional da Palavra
  RD <= mem(idx + 3) &
        mem(idx + 2) &
        mem(idx + 1) &
        mem(idx)
        when idx <= 124 else (others => '0');
end architecture;

Análise do Waveform: Estágio de Carga Inicial

Slide 12 / 18

🔍 Início da Simulação Funcional

out_pc inicia marchando linearmente em passos estáveis de 4 bytes (endereços 0, 4, 8, 12) a cada intervalo cíclico de clock. O sinal out_RegWrite opera ativado em nível lógico alto ('1') durante os ciclos iniciais de preparação, liberando a carga das constantes. Ao acessar as primeiras palavras contíguas mapeadas na RAM, os registradores out_reg_t2 (MAX) e out_reg_t3 (MIN) absorvem instantaneamente o valor inicial **15**.

Análise do Waveform: Convergência do Laço

Slide 13 / 18

🔄 Varredura Ativa do Vetor

O ponteiro de varredura out_reg_t0 incrementa de forma contínua, assumindo os endereços físicos das células: 4, 8, 12, 16, 20, 24, 28, 32. O barramento de dados out_reg_t5 espelha os elementos carregados linha por linha: **42, 4, 89, 23, 7, 55, 12**. A ALU customizada opera em ciclo único: t2 (MAX) atualiza-se para 42 e fixa-se no pico **89**, enquanto t3 (MIN) sofre uma queda imediata estabilizando em **4**.

Análise do Waveform: Escrita síncrona na RAM

Slide 14 / 18

💾 Escrita e Consolidação dos Extremos

O ponteiro atinge o valor limite de parada (32) e o desvio condicional do beq quebra o laço. A flag de gravação out_MemWrite satura em nível alto ('1'). No ciclo de escrita do máximo, a saída da ALU aponta para o endereço físico 32 e out_WD descarrega o valor **89** para a RAM. No ciclo imediatamente consecutivo, a ALU altera o ponteiro para 36 e out_WD injeta o valor mínimo **4**, assegurando a persistência completa dos dados.

Parametrização Temporal do Experimento

Slide 15 / 18

⏳ Definição do Intervalo de Simulação

O tempo total necessário para simular e visualizar toda a dinâmica estrutural do hardware no simulador foi projetado multiplicando o número de ciclos totais pelo período estabelecido:


  • Contagem de Instruções Dinâmicas (N): 52 instruções.
  • Como a CPU é Monociclo, 52 instruções equivalem a exatamente 52 ciclos de clock.
  • Período de Clock adotado (Tc): 800 ps (Frequência de 1.25 GHz).

🧮 Modelagem Analítica do Tempo Total

Aplicando a equação regulamentar de desempenho da CPU fornecida na disciplina:

Texec = N * CPI * Tc


Texec = 52 * 1 * 800 ps = 41.600 ps

⏱️ Conclusão Prática: A simulação funcional exige uma janela de tempo mínima de 41,6 ns para exibir a gravação do último elemento na memória RAM.

Validação Métrica de Desempenho

Slide 16 / 18

📊 Análise do CPI Médio do Sistema

Por tratar-se de uma arquitetura estrutural baseada em um ciclo de processamento estritamente Monociclo (Single Cycle), cada instrução gasta exatamente um pulso completo de clock para trafegar pelo caminho de dados:


CPI Médio = 1.00

🚀 Redução do Caminho Crítico

Como o CPI é fixo em 1 no hardware monociclo, a única forma de otimizar o tempo total de CPU (Tempo de CPU) é reduzindo drasticamente a quantidade de instruções que o compilador precisa gerar para resolver o problema:


Tempo de CPU = Contagem de Instrucoes * CPI * Periodo do Clock

Demonstração Matemática: Contagem de Instruções (N)

Slide 17 / 18

❌ Cenário 1: MIPS Padrão (Nativo)

Mnemônicos executados dinamicamente com base na ROM original:


  • Inicialização (4 inst.): addi, addi, lw, lw
  • Laço Dinâmico (7 elementos):
  • • Elementos com troca (42, 4, 89): 9 inst. cada
    (addi + beq + lw) + (slt + beq + add) + (slt + beq) + j
  • • Elementos sem troca (23, 7, 55, 12): 8 inst. cada
    (addi + beq + lw) + (slt + beq) + (slt + beq) + j
  • Última Passagem (2 inst.): addi, beq (desvia para o fim)
  • Escrita e Fim (4 inst.): addi, sw, sw, j

N = 4 + (3 * 9) + (4 * 8) + 2 + 4 = 69 instruções

✔️ Cenário 2: Nosso MIPS (Com max/min)

Mnemônicos executados após a otimização estrutural da ALU:


  • Inicialização (4 inst.): addi, addi, lw, lw
  • Laço Dinâmico (7 elementos):
  • • Corpo Fixo do Loop: 6 inst. por iteração
  • addi + beq + lw + max + min + j
  • • Total do laço ativo: 7 elementos * 6 inst. = 42 inst.
  • Última Passagem (2 inst.): addi, beq (desvia para fora)
  • Escrita e Fim (4 inst.): addi, sw, sw, j

N = 4 + 42 + 2 + 4 = 52 instruções

📉 Redução real de 69 para 52 mnemônicos executados. Economia matemática de 24,6% no tempo total de execução (Texec).
